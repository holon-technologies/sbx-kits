# Security Policy

## Reporting a vulnerability

Do not open a public issue for vulnerabilities or exposed credentials. Use the
repository's private GitHub security advisory form so the maintainers can assess
and remediate the report before disclosure.

Include the affected kit and release, reproduction steps, security impact, and
any suggested remediation.

## Supported releases

Only the most recent tagged release is supported while Docker Sandbox kits remain
experimental. Consumers should pin a tag, commit, OCI version, or OCI digest.
